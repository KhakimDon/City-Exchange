# Настройка локального бэкенда для работы с Netlify

## Вариант 1: Использование ngrok (Рекомендуется)

### Шаг 1: Установите ngrok
Скачайте и установите ngrok с https://ngrok.com/

### Шаг 2: Запустите локальный бэкенд
```bash
cd Backend
python3 manage.py runserver 0.0.0.0:8000
```

### Шаг 3: Создайте туннель с ngrok
В новом терминале:
```bash
ngrok http 8000
```

Вы получите URL вида: `https://xxxx-xx-xx-xx-xx.ngrok-free.app`

### Шаг 4: Обновите ALLOWED_HOSTS
В `Backend/config/settings.py` добавьте ngrok домен:
```python
ALLOWED_HOSTS = ['xxxx-xx-xx-xx-xx.ngrok-free.app', 'localhost', '127.0.0.1']
```

### Шаг 5: Создайте .env файл на фронтенде
В папке `Frontend` создайте файл `.env`:
```
VITE_API_BASE_URL=https://xxxx-xx-xx-xx-xx.ngrok-free.app
```

### Шаг 6: Пересоберите и задеплойте фронтенд на Netlify

## Вариант 2: Использование localtunnel (Альтернатива)

### Установка:
```bash
npm install -g localtunnel
```

### Запуск:
```bash
lt --port 8000
```

Получите URL и используйте его как в варианте 1.

## Вариант 3: Настройка роутера (Для постоянного доступа)

Если у вас есть статический IP и доступ к роутеру:
1. Настройте порт-форвардинг на порт 8000
2. Используйте ваш внешний IP адрес
3. Обновите ALLOWED_HOSTS с вашим IP

## Важно:
- При каждом перезапуске ngrok вы получите новый URL - нужно обновлять настройки
- Для постоянного URL можно использовать платную версию ngrok
- Убедитесь, что бэкенд запущен на `0.0.0.0:8000` (не localhost), чтобы принимать внешние подключения
